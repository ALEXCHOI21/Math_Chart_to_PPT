## 1. 개요 및 목적
본 보고서는 Alex님의 비즈니스 목표인 **"수학 그래프 이미지의 자동화된 PPT 변환 서비스"**를 안정적으로 운영하기 위한 오늘자 시스템 디버깅 및 파이프라인 고도화 내역을 담고 있습니다. 플랫폼 종속성(Vendor Lock-in)을 배제하고 유지보수 및 이식이 용이한 범용적인 구조로 재정비하였습니다.

## 2. 주요 해결 과제 및 디버깅 내역

### 2.1. PPT 변환 모듈 (`converter.py`) 상수 참조 오류 수정
- **이슈 사항**: PPT 슬라이드 내 직선(축 및 선)을 그릴 때, `python-pptx` 라이브러리의 존재하지 않는 속성인 `MSO_SHAPE.STRAIGHT_CONNECTOR`를 참조하여 예외(`AttributeError`)가 발생함.
- **해결 방안**: 연결선은 일반 Shape가 아닌 전용 Connector 객체 속성을 요구하므로, 코드를 `MSO_CONNECTOR.STRAIGHT`로 일괄 교체하고, 최상단에 올바른 열거형을 `import` 하도록 수정하였습니다.

### 2.2. Gemini AI 데이터 파싱 구조 방어 (Edge Case Detection)
- **이슈 사항**: Gemini 모델에서 추출한 그래프 JSON 데이터가 때때로 단일 `dict`가 아닌 `list` 형태로 반환되어 `'list' object has no attribute 'get'` 파싱 에러가 발생함.
- **해결 방안**: `converter.py`의 `create_ppt` 메서드 진입점에 반환 데이터가 `list`일 경우 첫 번째 요소를 자동으로 파싱하도록 **방어 로직(Fallback Logic)**을 추가하여 안정성을 확보했습니다.

### 2.3. SDK 런타임 호환성 확보 및 네트워크 타임아웃 회피
- **이슈 사항**: 구글 API 통신 중 Windows OS 특유의 `asyncio` DNS 해상도 충돌로 인한 타임아웃 발생 확인.
- **해결 방안**: 이전 세션에서 적용한 바와 같이 동기 함수를 비동기 스레드 풀에서 실행시키는 래핑(Wrapping) 기법을 통하여, Windows 환경의 `ProactorEventLoop` 충돌 문제를 우회 처리 완료했습니다.
