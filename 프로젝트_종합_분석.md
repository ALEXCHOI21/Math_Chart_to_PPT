## 1. 목표 분석 (Goal Analysis)

### 1.1 핵심 비즈니스 목표
본 프로젝트의 본질적 목표는 **「수학 교육 콘텐츠 자동화 플랫폼」** 구축입니다.

| 항목 | 상세 |
|------|------|
| **최종 산출물** | 수학 그래프 이미지 → PNG(배경투명) + 편집 가능 PPT |
| **타겟 사용자** | 수학 강사, 교재 출판사, 온라인 교육 플랫폼 |
| **핵심 가치 제안** | 수작업 30분 → 자동화 1분 (98% 공수 절감) |
| **수익화 모델** | B2B 건당 과금 (이미지당 $5~10) 또는 구독형 |
| **차별점** | AI 기반 자동 분석 + 투명 배경 PNG + 편집 가능 PPT |

### 1.2 프로젝트 스코프
```
입력 → AI 분석 → 데이터 추출 → PPT 생성 → 다운로드
       (Gemini) (좌표축/곡선) (python-pptx) (GraphEngine)
```

**처리 가능한 그래프 타입**:
- 좌표축 있는 2D 그래프 (선형, 곡선, 함수)
- 제목, 범례, 축 레이블 포함 그래프
- 단일 또는 다중 곡선 그래프

---

#### 2.1.2 Backend (API 서버)
**위치**: `backend/main.py` (FastAPI)  
**기술 스택**: FastAPI + Uvicorn + Google Gemini API  

**주요 엔드포인트**:
```
POST /upload → 이미지 업로드 & AI 분석 & PPT 생성
GET /download/{file_id} → PPT 파일 다운로드
```

**모듈별 역할**:
| 모듈 | 책임 | 상태 |
|------|------|------|
| `ai_analyzer.py` | Gemini API 호출, JSON 파싱 | ✅ 안정 |
| `converter.py` | JSON → PPTX 변환 | ✅ 안정 (최근 수정됨) |
| `png_generator.py` | PNG 생성 (배경 투명화) | ✅ 안정 |

**평가**: ✅ **운영 안정성 확보됨**
- ✅ 최근 3개 critical 버그 수정 완료
- ✅ Edge case 방어 로직 추가 (list vs dict 파싱)
- ✅ Windows asyncio 타임아웃 해결
- ⚠️ 데이터베이스 미보유 (임시 파일만 사용)
- ⚠️ 인증/인가 메커니즘 없음

### 3.2 기술적 개선 권장사항

#### 3.2.1 아키텍처 최적화
현재 상태:
```
Frontend (GitHub Pages) ↔ Backend API (단일 서버)
                         └─ Gemini API
```

개선안:
```
Frontend (CDN) ↔ API Gateway ↔ Worker Services
   (Cloudflare)   (AWS/Azure)  ├─ Graph Analyzer
                               ├─ PPT Generator
                               ├─ PNG Converter
                               └─ Cache Layer (Redis)
                  ↓
              Database (PostgreSQL)
              Storage (S3/GCS)
```

**이점**: 확장성, 병렬 처리, 캐싱

#### 3.2.2 데이터베이스 추가
현재: 임시 파일만 사용  
개선: PostgreSQL 추가
```sql
-- 사용자
CREATE TABLE users (
    id UUID PRIMARY KEY,
    email VARCHAR UNIQUE,
    api_quota INT DEFAULT 0,
    created_at TIMESTAMP
);

-- 변환 히스토리
CREATE TABLE conversions (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    input_file_url VARCHAR,
    output_pptx_url VARCHAR,
    output_png_url VARCHAR,
    status ENUM('pending', 'completed', 'failed'),
    created_at TIMESTAMP,
    processing_time_ms INT
);
```

#### 3.2.3 CI/CD 파이프라인
현재: 수동 배포  
개선: GitHub Actions
```yaml
name: Deploy Backend
on: [push]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Run tests
        run: pytest
      - name: Deploy to Render
        run: curl -X POST ${{ secrets.RENDER_WEBHOOK }}
```
